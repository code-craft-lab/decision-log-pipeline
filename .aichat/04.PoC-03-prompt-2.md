# Zadanie
Przygotuj Proof of Concept (PoC) serwisu „OPA Decision Log Pipeline” spełniającego tylko kluczowe założenia MVP:

1. Udostępnij endpoint `POST /decision-logs` (Spring WebFlux, Kotlin 2.x, Java 21).
2. Endpoint przyjmuje **JSON array** z decyzjami OPA (brak dodatkowej walidacji pól).
3. Uwierzytelnienie pojedynczym kluczem API przekazywanym w nagłówku `X-API-Key`. Jeśli klucz niewłaściwy → HTTP 401.
4. Po odebraniu batcha **natychmiast** opublikuj surowy JSON array do tematu Apache Kafka `opa.decisions.raw` (dowolny klucz partycji, brak chunkingu).
5. Zwróć HTTP 201 po poprawnym zapisie do Kafka; w razie błędu zwróć HTTP 500.

Zakres PoC wyklucza:
- zapisy do S3, OpenSearch, DLQ,
- metryki, monitoring, chunking > 1 MB,
- zarządzanie wieloma kluczami API,
- CLI/UX, re-process DLQ, testy wydajności itp.

# Stos technologiczny
- Kotlin 2.x + JVM 21
- Spring Boot 3.5 (WebFlux, Spring Security)
- Kafka (producer z Spring for Apache Kafka)
- Gradle 8.8
